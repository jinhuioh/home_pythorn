예외처리(Exception)
------------------
코드에 중간에 에러가 있으면 프로그램 실행은 중단되어버림.
중단시키지 않고, 에러가 발생한 부분만 실행을 시키지 않으면서
프로그램 실행을 계속 할 수 있게 하는 것.
에러나는 부분만 실행의 예외로 두고,
다른 부분의 시행은 계속 하겠다고 처리할 수 있음

try:#필수
    에러 가능성이 있는 코드 블록
except:#필수(자바의 catch)
    에러가 났을 때 어떻게 처리할지 내용을 코드함
    catch외에 이어지는 코드는 처리잘 됨.

finally: #옵션으로 써도 되고 안써도 된다.
    에러가 나는지 여부와 상관없이 반드시 실행 코드 블록

이어지는 코드
이어지는 코드
이어지는 코드~~여기는 실행이 잘 됩니다.


강제성!!(예외처리를 꼭 해야할까?)
-----------------------------
-일반적인 경우에는 프로그래머의 선택
-특별한 경우에는 반드시 예외처리를 해야함
 안하면 절대 안됨!!!!!꼭 예외처리를 해줘야하는 경우가 있다.
-언제든지 에러가 발생할 수 있는 위험을 가지고 있는 경우
1) 외부 자원을 연결하는 경우(cpu,file)
2)네트워크 연결(네트워크가 끊어질가능성이있으니까)(체팅프로그램)
3)db연결
